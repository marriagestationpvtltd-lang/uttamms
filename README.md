# Marriage Station App – System Guidelines

## 🎯 Objective

This project must be maintained and improved to a production-level matchmaking platform without breaking existing features.

---

## 🧠 Core System Flow

VERIFY → PACKAGE → REQUEST → ACCESS

---

## 🔐 Access Rules

### 1. Verification

* User must verify all required documents
* No partial verification allowed

### 2. Package

* Required only to send requests
* Does NOT grant direct access

### 3. Request

* Required for:

  * Chat
  * Photo view
  * Audio/Video call

---

## 💬 Chat Rules

* Chat screen → full message
* Conversation list → masked preview (free users only)
* No upgrade loop bugs

---

## 📄 Document Rules

* Only required documents shown
* One document per condition
* Verified documents locked
* Rejected documents re-uploadable

---

## ⚡ Performance Rules

* No duplicate API calls
* Optimize state management
* Fast loading UI
* Efficient image handling

---

## 🎨 UI/UX Rules

* No broken layouts
* No overlapping buttons
* Smooth keyboard behavior
* Clean, simple design

---

## 🔒 Security Rules

* No sensitive data exposure
* All inputs validated
* No feature bypass allowed

---

## 🧩 Code Rules

* No duplicate logic
* Use centralized state management
* Maintain clean architecture
* Write maintainable code

---

## ⚠️ Strict Warning

* Do NOT break working features
* Do NOT change API unnecessarily
* Always test after changes

---

## 🚀 Final Goal

A fast, secure, scalable, and professional matchmaking application.
